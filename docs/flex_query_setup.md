# IBKR Flex Query Setup — Step-by-Step

This document covers the one-time setup required before **🔄 Update Trades** works in the dashboard.

---

## 1. Enable Flex Web Service

**Portal:** Manage Account → Settings → Account Settings → Flex Web Service

1. Enable Flex Web Service for your account.
2. Click **Generate Token** and copy the token — you only see it once.
3. Add it to your `.env` file:
   ```
   TOKEN=<paste token here, no quotes or spaces>
   ```

The token must be generated by the **same portal user** who creates the Flex Query below.
A mismatched owner returns error 1004. An invalid token value (copy-paste whitespace) returns error 1015.

---

## 2. Create the Flex Query

**Portal:** Performance & Reports → Flex Queries → Activity → Create

| Setting | Value |
|---|---|
| **Query Name** | `TradeHistory` (or any name) |
| **Format** | XML |
| **Period** | Last 365 Calendar Days |
| **Date Format** | YYYYMMDD |
| **Time Format** | HHMMSS |
| **Date/Time Separator** | Semicolon (`;`) |

> The period must be a saved rolling period (e.g. "Last 365 Calendar Days"). The Custom Date Range option only works for manual portal downloads and does NOT apply to API calls — the API ignores it and returns no data.

---

## 3. Add the Three Required Sections

### Section 1 — Trades

Enable the **Trades** section. Then expand it and check the following individual fields:

| Portal label | XML attribute | Used for |
|---|---|---|
| Account ID | `accountId` | Account labelling (US/SG) |
| Date/Time | `dateTime` | Trade timestamp |
| Trade Date | `tradeDate` | Date filtering |
| Asset Class | `assetCategory` | Filter OPT vs STK |
| Symbol | `symbol` | Per-symbol P&L |
| Underlying Symbol | `underlyingSymbol` | Wheel strategy grouping |
| Put/Call | `putCall` | Option direction |
| Strike | `strike` | Option strike |
| Expiry | `expiry` | Option expiry |
| Multiplier | `multiplier` | Contract value |
| Quantity | `quantity` | Position size |
| Trade Price | `tradePrice` | Fill price |
| Trade Money | `tradeMoney` | Gross fill value |
| Proceeds | `proceeds` | Net proceeds |
| IB Commission | `ibCommission` | Commission tracking |
| Net Cash | `netCash` | Cash impact |
| Realized P&L | `realizedPnl` | P&L calculation |
| Open/Close Indicator | `openCloseIndicator` | Closed-trade filter |
| Close Price | `closePrice` | Mark-to-market |
| Trade ID | `tradeID` | Deduplication (primary) |
| IB Exec ID | `ibExecID` | Deduplication (fallback) |
| IB Order ID | `ibOrderID` | Deduplication (fallback) |

**CRITICAL — Options sub-reports must all be OFF.** In the Trades section click the **Options** button and confirm every sub-report checkbox is unchecked:

- [ ] Symbol Summary — **OFF**
- [ ] Order — **OFF**
- [ ] Execution — **OFF**
- [ ] Closed Lots — **OFF**
- [ ] Wash Sales — **OFF**

Enabling any sub-report (especially Execution) changes the XML element tag from `<Trade>` to `<Execution>`. The parser looks only for `<Trade>` and finds nothing — the download returns 0 rows with no error.

---

### Section 2 — Cash Transactions

Enable the **Cash Transactions** section. Required fields:

| Portal label | XML attribute |
|---|---|
| Account ID | `accountId` |
| Date/Time | `dateTime` |
| Amount | `amount` |
| Currency | `currency` |
| Type | `type` |
| Description | `description` |

This section populates `flex_cash.pkl` — deposit and withdrawal markers on the performance chart.

---

### Section 3 — Equity Summary in Base Currency

Enable the **Equity Summary by Report Date in Base Currency** section. Required fields:

| Portal label | XML attribute |
|---|---|
| Report Date | `reportDate` |
| Total | `total` |

This section populates `flex_nav.pkl` — the daily consolidated NAV that drives the performance chart and all KPI calculations (TWR, Sharpe, drawdown).

> Multi-account exports contain one row per account per day. The parser sums them automatically to produce a single consolidated NAV per day.

---

## 4. Save the Query and Note the Query ID

After saving, the query appears in your Flex Queries list. The **Query ID** is shown next to its name (a 7-digit number such as `1511039`).

Add it to `.env`:
```
TRADES_FLEXID=<your query ID>
```

For multiple accounts each with their own query, separate with commas:
```
TRADES_FLEXID=1511039,1511040
```

---

## 5. Add Account IDs to `.env`

```
US_ACCOUNT=U12345678
SG_ACCOUNT=U87654321
```

These are used to label trades in the history as "US" or "SG". Trades from accounts not in this list are still imported; the raw IBKR account ID is preserved as-is.

## 7. One-Time XML Bootstrap

The Flex API returns at most 365 days of history. For multi-year history, download one XML per year from the portal manually:

1. **Portal:** Performance & Reports → Flex Queries → `TradeHistory` → Run
2. Change **Date Range** to Custom, set **From** = Jan 1 YYYY, **To** = Dec 31 YYYY
3. Click **Run** and download the XML
4. Rename the file `2021.xml`, `2022.xml`, etc. (one file per year)
5. Save all files to `data/master/`

Repeat for each year back to the start of your trading history.

Then rebuild the pickle from these files:
```bash
uv run python scripts/update_trades.py --xml-only
```

After the initial bootstrap, `--xml-only` is no longer needed. The dashboard's **🔄 Update Trades** button (and `scripts/update_trades.py` with no flags) will top up via API automatically. Re-run the XML bootstrap only if you need to pull a fresh full-history XML from the portal (e.g. after a corporate action restates old trades).

## 8. Verify the Setup

```bash
uv run python scripts/diagnose_flex_api.py
```

This script calls the API, prints the raw XML structure, and gives targeted fix instructions. Common outcomes:

| Output | Cause | Fix |
|---|---|---|
| `OK: Found N <Trade> records` | Working correctly | None |
| `WARN: Data elements present but none are <Trade>` | Sub-report option is ON | Uncheck all Options sub-reports in portal |
| `WARN: <Trades> section present but 0 records` on weekend | Weekend API freeze | Wait until Monday |
| `FAIL: ErrorCode 1004` | Token/query owner mismatch | Regenerate token as the query owner |
| `FAIL: ErrorCode 1015` | Invalid token value | Re-copy token from portal (no spaces) |
| `FAIL: ErrorCode 1006` | Flex Web Service not enabled | Enable in portal Account Settings |
| `FAIL: <Trades> section missing` | Trades section not added | Add Trades section to query |

## 9. Regular Maintenance

| Frequency | Action |
|---|---|
| Weekly | Click **🔄 Update Trades** in dashboard (or `uv run python scripts/update_trades.py`) |
| After each `build.py` | Click **🗓 Identify Weeklies** to refresh `symbol_categories.pkl` |
| Annually | Download a new year XML from portal and run `--xml-only` to merge it |
| Token expires | Regenerate in portal Settings → Flex Web Service; update `.env` |
