# IB Monitor — IBKR Portfolio Dashboard

## Start

```bash
uv run ibd
```

## What it does

Real-time portfolio risk monitor and options strategy manager for IBKR accounts. Connects via IB Gateway/TWS, streams live positions, Greeks, and account metrics, and automates the weekly wheel-strategy cycle (sow → cover → reap → protect) for S&P 500 options.

---

## Layout

A **single scrolling page** (no tabs): header bar → **KPIs** expander → **Actions** pipeline expander → **Ask AI** expander → one **global filter bar** (Symbol · C/P · State · ITM) that drives every panel below → content expanders. The Actions pipeline runs left-to-right: **Generate OHLCs → Refresh Flex → Run Backtest → Generate Orders → Execute Orders**. Styled with a Claude-inspired theme (clay accent, ivory canvas, serif headings) that adapts to light and dark mode.

### Analysis panels
- **Cumulative Performance chart** — portfolio TWR vs SPY/QQQ from inception; USD deposits stripped via time-weighted return. SGD deposits appear as markers but are excluded from TWR math (no FX rates). Period buttons: Focus, MTD, 1M, 3M, YTD, 1Y, 3Y, All. Deposit/withdrawal markers from Flex history. KPI cards: Realized OPT P&L, Consolidated TWR, Alpha vs SPY, Max Drawdown, Sharpe Ratio. Chart extends to today even on weekends/holidays (live IB NLV shown).
- **Live positions** — filterable table of all current holdings with Greeks (delta, theta, vega per position). ITM Only and Weekly Only filters.
- **Portfolio treemap** — market-value composition by symbol.
- **Cover/Protect gaps** — assigned stock without a covered call, or unhedged positions needing a put.
- **Options P&L by underlying** — realized + unrealized breakdown per symbol.
- **Symbol Deep-Dive** — per-symbol trade history, win rate, profit factor, strategy fingerprint (CSP/CC/LP/LC), and BacktestExpert score.
- **Backtest scoring** — composite 0–100 score per symbol (sample density, expectancy, risk, robustness); verdict DEPLOY / REFINE / ABANDON.
- **OHLC candlestick** — Bollinger Bands + RSI for any symbol, sourced from `ohlc.pkl`.

### Orders panels
- **⚡ Actions expander** (top of page) — the full daily pipeline plus Clear Data, with status messages and live logs.
  - **Generate Orders** — runs `derive.py` as a subprocess; dashboard freezes CID=10 first. Button turns blue while running.
  - **Execute Orders** — submits derived orders via `execute.py`. `rich` progress bars go to the terminal; dashboard log shows counts only.
- **Suggested orders** — strategy tables generated by `derive.py`:
  - **Cover** — sell covered calls on assigned stock (monthly expiry for monthly-only symbols; weekly otherwise)
  - **Sow** — sell naked puts/calls on S&P 500 weeklies (monthly-only symbols excluded automatically)
  - **REFINE Overrides** — per-symbol `VIRGIN_PUT_STD_MULT` override table for REFINE-rated symbols; saves to `data/symbol_overrides.json`, picked up by `derive.py` on next run. Always visible (shows info message when no backtest results exist).
  - **Reap** — buy-to-close profitable open options (filtered to live positions via conId)
  - **Protect** — buy puts for downside hedging
- **Open orders** — live IBKR pending orders with status.

## Ask AI

Collapsible **Ask AI** expander near the top of the page. Queries the full live data context and returns answers in plain language. The provider defaults to `DEFAULTAI` from `config/snp_config.yml`.

**Data the AI can access:**

| Source | What the AI knows |
|---|---|
| Live positions | Current holdings with right/strike/expiry/Greeks (AUTHORITATIVE — not inferred from history) |
| Account metrics | NLV, cushion (ExcessLiquidity/NLV), init margin, buying power, available funds, Leverage-S |
| Greeks | Portfolio dollar delta, theta, vega |
| Trade history | 6+ years of closed OPT trades — global stats (win rate, profit factor, total P&L, **total commissions**), per-symbol breakdown, and most recent 200 trades chronologically |
| Backtest scores | Per-symbol composite score, verdict, sub-scores, flags (top-80 by trade count) |
| OHLC stats | Last price, 20/90-day return, 52-week position, MA trend, HV20 for held + top-traded symbols |
| OHLC price history | Monthly close prices (split-adjusted, last 24 months) per symbol |
| Consolidated NAV | Full month-end NAV history from ~2020; pre-computed KPIs: Sharpe ratio (full history + since Jan 2025), TWR full history, Max drawdown (full history + since Jan 2025) |
| SPY / QQQ benchmarks | Monthly closes from Jan 2020 — for multi-year return comparisons |
| Cash transactions | Deposits/withdrawals (last 2 years), dividends + interest by year |
| Suggested orders | Cover/Sow/Reap/Protect tables with expected premium totals |
| Symbol classification | Monthly-only vs weekly S&P 500 symbols (from `symbol_categories.pkl`) — answers "does X have weekly options?" |

**Example questions:**
- "What is my total portfolio delta and theta today?"
- "Which symbols have the worst backtest scores and why?"
- "How did my portfolio perform vs SPY in 2022?"
- "What covered calls should I sell this week?"
- "Is DVN a weekly or monthly options symbol?"
- "How much have I paid in commissions historically?"
- "What's my cushion and am I close to a margin call?"
- "Show me the 5 best and 5 worst trades of all time."
- "What is my Sharpe ratio and max drawdown since Jan 2025?"

**Providers:** DeepSeek (default), Gemini.
**Conversation:** 5-turn rolling window; `💬 N` button clears history.

## Configuration

Edit `config/snp_config.yml` for PORT, CID, MINCUSHION, MAX_DTE, and strategy parameters.

Key cover parameters:

| Key | Default | Description |
|---|---|---|
| `COVER_ME` | `true` | Enable covered call generation |
| `COVER_MIN_DTE` | `4` | Minimum DTE for covered call expiry search window |
| `COVER_STD_MULT` | `0.5` | Strike floor = undPrice + mult × sdev (1 std dev at DTE) |
| `COVXPMULT` | `1.2` | Expiry multiplier for cover DTE window |
| `COV_AGED_DTE` | `180` | Days since assignment after which covPrice switches from `max(avgCost, vol_based_price)` to `vol_based_price` only — earn income rather than waiting to recover cost basis on long-held positions. Assignment date is loaded from `flex_trades.pkl`. |

Set secrets in `.env`:

```
US_ACCOUNT=your_us_account_id
SG_ACCOUNT=your_sg_account_id
TOKEN=your_flex_web_service_token
TRADES_FLEXID=your_activity_flex_query_id
GEMINI_API_KEY=...       # for Gemini provider
DEEPSEEK_API_KEY=...     # for DeepSeek provider (default)
```

`TOKEN` and `TRADES_FLEXID` are required for **🔄 Update Trades**. See [docs/flex_query_setup.md](docs/flex_query_setup.md) for portal setup and the one-time XML bootstrap process.

## Batch pipeline (run from project root)

| Command | Purpose |
|---|---|
| `uv run python src/build.py` | Fetch qualified contracts, option chains, and volatilities from IBKR. When market is closed and IBKR IV/HV are unavailable, falls back to OHLC-based 30-day realized vol automatically — `derive.py` can generate orders at any hour. |
| `uv run python src/derive.py` | Generate optimal option orders (sow/cover/reap/protect) |
| `uv run python src/execute.py` | Submit derived orders to IBKR |
| `uv run python src/analyze.py` | Portfolio analysis |
| `uv run python src/fetch_ohlc.py` | Update OHLC history (yfinance primary, IBKR fallback) |
| `uv run python src/clear.py` | Clear data files (preserves `data/master/`) |

Add `--debug` to any script to show DEBUG output in terminal. DEBUG always goes to `log/<script>.log`.

**After each build:**
1. Run `build.py` → `derive.py`
2. Click **🗓 Identify Weeklies** (Analysis → Actions) to refresh `symbol_categories.pkl` — keeps monthly-only exclusion list current.

**Off-hours IV fallback:** `build.py` writes `data/build_summary.json` after every run. When IBKR doesn't return IV (frozen feed, outside market hours), it falls back first to IBKR HV (tick 104), then to 30-day realized vol computed from `data/master/ohlc.pkl`. The summary records `hv_fallback.count` and `hv_fallback.symbols`. The Cover/Sow/Reap expander headings in the dashboard show `(used hv for N symbols)` when this fallback is active.

## Flex Trade History

Three protected pickles in `data/master/`, all refreshed by **🔄 Update Trades**:

| Pickle | Contents |
|---|---|
| `flex_trades.pkl` | All closed trades — OPT + STK. Fields include `ibCommission`, `realizedPnl`, `proceeds`, etc. |
| `flex_cash.pkl` | Cash transactions — deposits, withdrawals, dividends, interest. |
| `flex_nav.pkl` | Daily consolidated NAV (US + SG summed). Powers the performance chart and Ask AI NAV context. |

Standalone refresh: `uv run python scripts/update_trades.py`
Rebuild from local XMLs only: `uv run python scripts/update_trades.py --xml-only`

## Symbol Classification

S&P 500 symbols are classified as **weekly** (chain gap <20 days) or **monthly-only** (gap ≥20 days). ~257 of 502 are monthly-only. Classification is stored in `data/master/symbol_categories.pkl`.

- `derive.py` uses it to exclude monthly-only symbols from weekly sow candidates.
- `derive.py` generates breakeven covered calls for monthly-only assigned stock (`df_monthly_cov.pkl`).
- Ask AI uses it to answer "does X have weekly options?"
- Refresh: Analysis tab → Actions → **🗓 Identify Weeklies** (after each build).

Standalone: `uv run python scripts/update_symbol_categories.py`
